Linux文件系统

Date 2013年09月04日 By masteryoda Category OS.

Linux文件权限

文件权限以10位字符表示，比如 '-rwxrwx---'

第一个字符代表这个文件是 “目录[d]/文件[-]/链接文件[l]等 ”。

之后9个字符分成三组，第一组为文件所有者的权限，第二组为同用户组用户的权限，第三组为其他非本用户组用户的权限。rwx分别代表权限可读/可写/可执行。

如果文件名以“ . " 开头，说明这是一个隐藏文件

权限对于文件

  • r(read): 可读取此文件的实际内容，如读取文本文件的文字内容等。
  • w(write): 可编辑，新增或是修改该文件的内容（但不包含删除该文件）。
  • x(eXecute): 文件是否可以执行。

权限对于目录

  • r(read contents in directory): 表示具有读取目录结构列表的权限， 可以查询该目录下的文件名数据。
  • w(modify contents of directory): 可写入， 表明具有更改该目录结构列表的权限，包括新建新的文件与目录， 删除已经存在的文件与目录（不论该文件的权限为何）， 将已存在的文件或目录进行重命名， 转移该目录内的文件或目录的位置。
  • x(access directory): 表明用户能否进入该目录成为工作目录。

改变文件属性与权限

  • chgrp: 改变文件所属用户组

    chgrp [-R] groupname dirname/filename ...

  • chown: 改变文件所有者

    chown [-R] username dirname/filename ...
chown [-R] username:groupname dirname/filename ...

  • chmod: 改变文件的权限

    可以分别用数字和符号來更改权限，每个权限用一个bit來保存—— r:4, w:2, x:1。 rwx权限就为4+2+1=7。

    chmod [-R] xyz dirname/filename ...

    使用符号改变权限时，用u,g,o分别代表user,group,others 3种身份，a代表all。 +, -, =分别代表加入，去除，设置权限。

    chmod [-R] [ugoa] [+1=] [rwx] dirname/filename ...

chmod u=rwx,go=rx .bashrc

chmod a+w .bashrc

Linux目录结构

FHS制定出来的四种目录特色为shareable，unshareable，static，variable

FHS所定义的三层主目录为/，/var，/usr

有五个目录必须与根目录放置在同一分区，/etc, /bin, /lib, /dev, /sbin

  • /bin: 单用户维护模式下还能被操作的命令，/bin下的命令可以被root与一般帐号使用。
  • /boot: 开机会用到的文件，包括Linux内核文件以及开机菜单/配置文件等。
  • /etc: 系统主要的配置文件都放在此目录下。
  • /home: 系统默认的用户主文件夹。
  • /lib: 开机时会用到的函数库以及在/bin或/sbin下面的命令会调用的函数库
  • /media: 放置可删除的设备，包括软盘光盘DVD
  • /mnt: 暂时挂载目录
  • /opt: 第三方软件放置于此
  • /root: 系统管理员的主文件夹
  • /sbin: 放在此目录下的为开机过程中所需要的命令，包括开机修复系统还原等
  • /proc: 虚拟文件系统，放置的数据都是内存之中的。
  • /sys: 也是一个虚拟文件系统，主要记录内核相关的信息。
  • /usr: UNIX Software Resource的缩写，所有系统默认软件的放置目录
  • /var: 主要针对常态性变动的文件，包括缓存/登录文件以及某些软件运行时产生的文件。
