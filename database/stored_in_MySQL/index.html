MySQL存储程序

Date 2013年09月06日 By masteryoda Category database.

MySQL支持把几种对象存放在服务器端供以后调用，包括：

  1. 存储函数（stored function）返回一个计算结果，该结果可以用在表达式里
  2. 存储过程（stored procedure）不直接返回一个结果，但可以用来完成一般的运算或生成一个结果集并传递给客户
  3. 触发器（trigger）与数据表相关联，当那个数据表被INSERT，DELETE，UPDATE语句修改时，触发器自动执行
  4. 事件（event）根据时间表在预订时刻自动执行

存储函数

CREATE FUNCTION语句來创建，RETURNS 來指明返回值类型，BEGIN END來表明复合语句的起止，RETURN 语句來返回结果

CREATE FUNCTION count_born_in_year(p_year INT)
RETURNS INT
READS SQL DATA
BEGIN
RETURN (SELECT COUNT(*) FROM president WHERE YEAR(birth)=p_year);
ENDS

调用时和调用内建函数一样

count_born_in_year(1913)

存储过程

不返回值，因此没有RETURNS和RETURN语句

CREATE PROCEDURE show_born_in_year(p_year INT)
BEGIN
    SELECT first_name, last_name,birth, death
     FROM president
     WHERE YEAR(birth)=p_year;
END

无法用在表达式里，只能通过CALL语句來调用

CALL show_born_in_year(1908);

一个过程可以生成多个结果集，他们将依次被发送到客户

  • 存储函数有一个限制：不允许对调用本函数的语句正在读或写的数据表进行修改。

  • 存储过程通常没有这个限制，但如果他们是从存储函数里被调用，则要遵守这个限制。

存储过程的参数分为IN，OUT，INOUT三种

  • IN是调用者把一个值传递给过程
  • OUT则相反，过程把一个值附给OUT，这个值在过程返回后可以由调用者使用
  • INOUT允许调用者向过程传递一个值，然后再取回一个值

  • 默认参数都是IN

    CRRATE PROCEDURE count_students_by_sex (OUT p_male INT, OUT p_female INT)
BEGIN
    SELECT COUNT(*) FROM student WHERE sex='M' INTO p_male;
    SELECT COUNT(*) FROM student WHERE sex='W' INTO p_female;
END
---
CALL count_students_by_sex(@mcount, @fcount);
SELECT 'Number of male students:', @mcount;

触发器

与特定数据表相关联的存储过程，当那个数据表被INSERT，DELETE，UPDATE语句修改时，触发器自动执行，可以被设置为在这集中语句处理每个数据行之前或之后触发。

CREATE TRIGGER trigger_name         # the trigger name
    {BEFORE | AFTER}                # when the trigger activates
    {INSERT | UPDATE | DELETE}      # what statement activates it
    ON tbl_name                     # the associated table
    FOR EACH ROW trigger_stmt;      # what the trigger does

trigger_stmt是触发器的语句体部分，其中可以使用NEW.col_name语法來引用将由INSERT或UPDATE语句插入或修改的新数据行，OLD.col_name可以使用将由DELETE或UPDATE语句删除或修改的老数据行里的数据

CREATE TRIGGER bi_t BEFORE INSERT ON table_t
    FOR EACH ROW BEGIN
     SET NEW.dt = CURRENT_TIMESTAMP;
     IF NEW.percent < 0 THEN
          SET NEW.percent=0;
     ELSEIF NEW.percent>100 THEN
          SET NEW.percent=100;
     END IF;
    END

事件

事件是一个与时间表相关联的存储程序，时间表用来定义事件发生的时间/次数以及何时消失。

默认情况下事件调度器不会运行，如果想使用事件，必须先启动事件调度器，把以下语句添加到选项文件中

[mysqld]
event_scheduler=ON

如果想在系统运行时停止或启动事件调度器，可以通过改变event_scheduler系统变量的值來达到目的

SET GLOBAL event_scheduler = OFF/ON;    or 0/1

创建事件：

CREATE EVENT expire_web_session
    ON SCHEDULE EVERY 4 HOUR
    DO
     DELETE FROM web_session
     WHERE last_visit < CURRENT_TIMESTAMP - INTERVAL 1 DAY;

EVERY n interval子句用来给出事件定期执行的时间间隔，其后还可以用STARTS datetime 和ENDS datetime选项给出事件第一次和最后一次执行的时间。

DO语句负责定义事件的语句体部分。

如果想定义一个只执行一次的事件，可以使用AT调度而不是EVERY

禁用事件和启用事件：

ALTER EVENT event_name DISABLE/ENABLE;
